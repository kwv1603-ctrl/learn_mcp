# AntiGravity 自动驾驶准则 - Finance Learn MCP 项目

## 1. 报告存储规范 (CRITICAL)
- **禁止使用隐藏目录**：严禁将分析结果、报告、图片保存至 `.gemini/` 或其他系统隐藏文件夹。
- **默认报告路径**：所有生成的报告必须统一保存至 `./reports/` 目录下。
- **文件名规范**：使用 `symbol_analysis_YYYYMMDD.md` 格式。

## 2. 技能与工具使用规范
- **核心入口**：本项目的结构化金融数据、估值、巴菲特评分、技术面、同行对比、宏观上下文、报告准备度与报告校验，必须优先通过 `./mega_finance_bridge.py` 暴露的 MCP 工具进行调用；工具清单、参数转发与 MCP 协议层保留在入口文件，具体工具实现集中在 `./finance_mcp_tools.py`。
- **职责边界**：`mega_finance_bridge.py` 不提供通用 Web 搜索。行业 KPI、官方披露、监管事件、年报附注等非结构化证据，使用当前运行环境可用的搜索/浏览能力获取，并在报告中显式列出来源或标注未取得。
- **禁止冗余**：不要为每次分析创建新的 runner 脚本；允许通过 MCP `tools/call`、命令行 one-liner 或直接调用 bridge handler 完成一次性数据读取。
- **本地 Skill 优先**：在处理估值、技术面和巴菲特评分时，必须优先使用工作区 `skills/` 目录下的 Python 实现，而不是系统内置的默认逻辑。

## 3. 分析流程
0. **首先**完整阅读根目录下的 `report_rules.md` 模版（参见 §5 核心指令），确保对 13 大模块及各子模块要求形成完整认知。
1. 调用 `run_report_readiness_check` 获取报告前置数据状态与缺失项。
2. 调用 `mega_finance_bridge.py` 获取量化结果，并使用环境可用搜索能力补齐行业 KPI、官方披露、监管事件和同行证据。
3. 结合量化结果、外部证据与巴菲特框架资料进行综合推理。
4. 直接在 `reports/` 下更新或创建 Markdown 报告。
5. 报告生成后调用 `validate_report` 做结构、口径和硬规则校验；若未通过，必须修正或在报告末尾显式说明未通过原因。

## 4. 输出规范
- **严禁垃圾文件 (HARD RULE)**：严禁为单次分析新增中间 runner 脚本、临时代码文件、测试文件或人工 scratch 数据文件。历史 `scratch/` 文件只作为遗留材料保留，不作为后续新增输出路径。
- **执行方式**：数据获取、处理和逻辑执行应通过 MCP 工具、命令行 one-liner 或直接 handler 调用完成；不得通过新增临时脚本规避 bridge 接口。
- **直给报告**：针对分析请求，直接输出最终的 Markdown 报告。

## 5. 报告结构与深度规范 (STANDARD)
- **核心指令 (CRITICAL)**：在生成任何分析报告前，**必须首先完整阅读并遵循** 根目录下的 [report_rules.md](./report_rules.md) 中的 Markdown 模版。
- **一致性要求**：每份分析报告必须严格按照 `report_rules.md` 定义的 13 个大模块进行组织。生成后必须逐子模块对照模版检查：
  1. 每个子模块标题是否存在（不得跳过或合并）
  2. 表格类子模块的**每一行**是否被填写（即使判定为 N/A 也需保留行并标注理由）
  3. 计算类子模块的代码块是否使用了模版指定的公式（禁止自行替换公式或跳过计算步骤）
  严禁简化标题或删除核心量化指标。
- **模版对齐**：生成的报告与模板做最终对齐，确保完全一致，梳理其中的逻辑，务必保证逻辑自洽。

## 6. 深度质疑协议 (CRITICAL — "巴里克教训")

> **背景**: 巴里克黄金 (B) 的首次分析中，量化评分 78% 看似优秀，但完全遗漏了"产量 6 年下降 40%"的结构性风险。报告变成了"数字好看 → 结论乐观"的肤浅循环。以下协议确保此类问题不再发生。

**在获取量化评分后、写报告之前，必须完成以下步骤：**

1. **独立定性研究 (MANDATORY)**：通过当前运行环境可用的搜索/浏览能力完成以下调研，并在报告中列出来源或显式标注未取得：
   - 公司核心 KPI 的 **5 年趋势** (如产量、用户数、门店数、订单量、ASP 等行业特定指标)
   - 利润增长到底是 **"量驱动"还是"价驱动"**（量价分离分析）
   - 近 2 年是否有 **资产出售、停产、监管冲击** 等重大事件
   - 行业内是否有 **同行表现更好** 的替代标的

2. **"魔鬼辩护人"检查 (MANDATORY)**：对量化评分的每个高分维度，必须追问：
   - 这个高分是由 **经营实力** 驱动，还是由 **外部因素** (如商品价格、汇率、政策) 驱动？
   - 如果外部因素逆转，这个维度会变成什么评分？
   - 量化模型是否遗漏了某个 **结构性变量** (如产量下降、客户流失、技术替代)？

3. **评分修正 (WHEN APPLICABLE)**：当发现量化模型未覆盖的重大风险时：
   - 必须在报告中 **明确标注** 原始评分与修正后评分
   - 用 ⚠️ 符号标注每一条被外部因素扭曲的维度
   - **禁止** 将高量化评分直接等同于"优秀投资标的"

> **注**: 行业特定 KPI 检查清单与利润纯度三问已内置于 `report_rules.md` 模版的 §6.5 和 §6.6 中，无需在此重复。

## 7. 熊市与左侧择时纪律 (CRITICAL — "防接飞刀教训")

> **背景**: 纯粹的价值投资极易在熊市深跌中陷入“接飞刀”陷阱（Value Trap 或流动性杀跌）。当标的处于明显空头趋势时，单纯的“低估 = 买入”会导致严重的左侧回撤，击穿资金防线。

**在输出交易策略前，必须将【估值】与【技术面趋势】进行二维交叉验证：**

1. **趋势鉴别**：检查基本技术面因素。如果均线空头排列、处于明显下降通道、或是显著的“熊市（Bear Market）”走势。
2. **强制左侧预警**：即使基本面与估值得出【严重低估】结论，也 **严禁** 直接给出无条件的“买入 (Buy)”全仓评级。必须在报告第1章及第13章强调 **⚠️ 熊市左侧预警 (Bear Market / Left-side Warning)**。
3. **纪律性操作建议**：在熊市行情下，必须将操作建议强制修改为以下两种之一：
   - **观望等待 (Wait & See)**：建议保持观望，不盲目入场，耐心等待均线走平、右侧突破等明确的技术面止跌信号后再介入。
   - **极左侧分批/逐步抄底 (DCA/Staged Buying)**：若具有极高安全边际且极度确信，只能建议留足现金，通过极低仓位（如总仓位 10%-20%）进行网格化/分批次左侧建仓，越跌越买，**严禁**建议一次性重仓建仓。

## 8. 术语精准度与量化失真人工纠偏 (CRITICAL)

**1. 严禁滥用“价值陷阱 (Value Trap)”标签：**
- **价值陷阱的统一定义**：仅适用于那些业务实质上处于长期结构性衰退、产品被淘汰、基本面持续恶化，但由于股价暴跌导致 PE/PB 指标看似极具吸引力的“劣质资产”。
- **风险溢价 (Risk Premium) 的区分**：对于基本面极其优异、护城河深厚的极品公司，如果其估值因地缘政治、政策打压等“外部尾部风险”而被压制，这叫**“存在系统性折价/地缘风险溢价”**，绝不可以称其为“价值陷阱”。

**2. 重资产与量化机器计分失真的强制纠偏：**
- 当自动化脚本（如 `mega_finance_bridge.py`）对晶圆代工、矿业、电信等需要维持庞大资本开支（CapEx）构建绝对技术壁垒的超级重资产公司打出极低的主人盈余（Owner Earnings）或内在价值分数时，**AI 必须人工介入识别失真**。
- 不得将剥离了维持护城河级 CapEx 导致的低分直接解读为“基本面垃圾”。必须在报告中引入专门的“评分质疑声明”，从其绝对的提价权（Pricing Power）和垄断地位纠正机器打分的盲区。

## 9. 数据保真与 A 股财报透视纪律 (CRITICAL — "零捏造原则")

> **适用范围**：以下纪律专为中国 A 股设计。分析港股/美股时，若 API 的 CapEx 数据与同行/历史常态一致无明显异常，可跳过附注强制搜索步骤。

由于部分通用 API（如 yfinance, akshare）在抓取中国 A 股现金流量表时，会将《购建固定资产支付的现金》与《购建无形资产支付的现金》合并为同一项大额 CapEx，导致主人盈余（Owner Earnings）被严重低估，必须严格遵守以下财务数据纪律：

1. **疑似合并项目的强制搜索**：如果遇到 A 股公司的 CapEx 与同行或历史常态相比极度异常，必须使用当前运行环境可用的搜索/浏览能力，精确搜索该公司当年年报的《财务报表附注》明细（如搜索引擎输入：`公司代码 购建固定资产支付的现金 202X`）。
2. **绝对零捏造 (ZERO HALLUCINATION)**：不管你通过推理觉得某个财务拆分数值“应该”是多少，**绝对不允许编造或通过比例倒推数据**。
3. **截面时间一致性**：必须保证报告内部所有引用的数据口径时间节点完全一致。严禁在同一张图表或公式中，把 2023 年的折旧与 2024 年的净利润混用，要么全部用 2024 年报数据，要么全部用 2025（即使是快报也需标注清楚）。
4. **缺失字段显性化**：如果动用了可用搜索/浏览能力依然无法在网上查到具体细分字段的官方数据，**必须**在报告的对应位置（如主人盈余计算公式处）显式写明 `未获取到结构化拆分数据` 或直接显示变量缺失，严禁在此处掩盖问题或用大数糊弄过关。
