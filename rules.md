# AntiGravity 金融分析 Agent 标准操作程序 (SOP)

## 1. 核心工作流 (The Workflow)
1. **预检**: 收到股票代码后，第一步**必须**调用 `run_report_readiness_check`。
2. **数据获取 (效率优先)**: 严格遵守：**本地数据库 > 自动 API > 手工搜索** 的优先级。**严禁**在第一步就去网上搜索年报原件。
3. **撰写报告**: 严格按照 `report_template.md` 的结构生成 Markdown 报告，存入 `./reports/` 目录下（命名：`symbol_analysis_YYYYMMDD.md`）。**严禁省略任何一章或表格中的任何一行。**
4. **强制校验 (Veto Gate)**: 报告生成后，**必须**调用 `validate_report` 工具进行结构和逻辑校验。如果校验失败（Status: fail），**必须**根据报错细节修复报告，直到校验通过。

## 2. 绝对不可触碰的高压线 (Zero Tolerance Rules)
- **禁止多余文件 (NO EXTRA FILES)**: 整个分析流程中，**只允许**生成最终的 Markdown 报告文件到 `reports/` 目录下。绝对禁止编写或生成任何多余的数据文件（如 .json、.csv）、测试脚本（.py）或其他临时文件。所有工具调用和数据处理必须在内存中完成。
- **零捏造 (ZERO HALLUCINATION)**: 如果自动化工具和搜索都找不到某个财务数据，**必须填入 N/A 或“该项缺失”**，绝对不允许用比例倒推或编造。
- **A 股 CapEx 陷阱**: 如果标的是 A 股（代码以 .SS / .SZ / .SH 结尾），由于 API 可能将无形资产合并入固定资产，**你必须人工调用搜索工具，查找该公司该年度的《财务报表附注》**，手动拆分 CapEx 并重新计算 Owner Earnings。
- **熊市择时纪律**: 如果在第 10 章（技术面）判定为熊市或下降通道，第 1 章和第 13 章的**评级上限锁定为“观望”或“持有”**，严禁建议直接买入，必须提出左侧预警。
- **一致性**: 报告内所有的财务计算（如计算 Owner Earnings），必须统一使用同一口径（合并口径或归母口径），并在公式中显式标明。

## 3. 深度分析与人工纠偏 (Deep Reflection)
- **警惕价值陷阱**: 低市盈率可能是因为行业衰退或宏观崩塌。如果发现基本面持续恶化或面临系统性地缘风险，必须在第 2 章和第 9 章进行“折价处理”或标记风险。
- **重资产公司的机器打分失真**: 如果是矿业、晶圆代工等重资产公司，机器给出的 Owner Earnings 可能会因为极高的维护性开支而拿低分。你必须从其绝对定价权和垄断地位出发，在第 6.1 章进行“人工纠偏”。
